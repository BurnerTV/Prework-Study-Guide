HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML stands for HyperText Markup Language.
  • HTML is used to create the structure or foundation of the webpage.
  • It is made up of elements that work together to provide content and basic framework.
  • Test elements, image elements, header elements, list elements.
  • elements are also used to divide the content on the page and to describe the content to browsers and other devs.
  • Meta-related elements, content division elements and semantic elements are used to divide and describe a page.
  • HTML framework uses angle brackets for opening and closing tags.
  • Head element is a container for metadata
  • Meta elements contain information about the page that is used by the browser.
  • The charset element sets character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • HTML will have an index.html file.

CSS

  • There are 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheet and external CSS style sheet.
  • Inline CSS is for styling an element directly in the HTML file. It applies to a specific HTML tag. Useful for quick and permanent changes but less flexible.
  • Inline style must be edited individually whenever you make a design change.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS stands for Cascading Style Sheets.
  • CSS is the decorative part of a webpage.
  • CSS will have a style.css file.
  • The margin property assigns px amount to top and bottom, while auto value to the left and right.
  • The border property assigns a border. Px, color and type all determined by the code used in CSS.
  • The box-shadow CSS property adds shadow effects around an element's frame.

Git

  • Git status: checks what branch we are currently on.
  • Git checkout -b branch-name: creates a new branch and switches to it.
  • ls shows all files and directories in current location.
  • cd followed by a location enters us into that specific folder or directory.
  • pwd command prints current directory into the terminal.
  • Cd .. allows us to go back 1 directory at a time.
  • mkdir followed by a name creates a brand new directory.
  • touch such as touch first-day.txt creates a new file in the terminal.
  • code . opens current folder in VS code.
  • git status checks current branch.
  • git add -A states that we want to add all changes.
  • git commit -m with a description tells us what we are committing.
  • git pull origin main tells the computer where the pull request is going.
  • git push origin feature/insertbranchyoureworkingonhere pushes the new branch to github for a pull request.
  • git checkout main and git pull origin main are used consistently to switch to the main branch and then sync your remote and local main branches.

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Avoid using var because all modern JS uses new syntax.
  • The const keyword allows us to create variables that can't be reassigned a value. Once declared, we can never reassign it.
  • let is another variable keyword we might use.
  • After the const keyword we give the variable a name, it must be a unique name because it's how we reference the info stored in the variable in our code.
  • The const on this particular webpage is topics so it looks like: const topics "HTML, CSS, Git, Javascript, CLI";
  • To access the string "HTML, CSS, Git, Javascript, CLI" we can simply refer to the variable's name, topics.
  • console.log outputs the value of topics to the web console because we added it as an arguement inside the parentheses.
  • There are multiple ways to access the web console, F12 while on the webpage is my preferred way.
  • Note that while the style.css is linked within the head element, the script.js file is linked within the body element.
  • JS uses if, else if and else statements. else if statements help us provide more than one condition in JS.
  • When the computer encounters an if statement in a JS file it will check to see if the condition we provide is "truthy."
  • If the condition isn't true or has a value that isn't truthy, the computer will ignore whatever is in between the curly brackets and move on.
  • An array is a single variable that holds a group of data. Typically the data in an array is all related in some way.
  • A for loop performs a task on each piece of the data using just a few lines of code.
  • To create an array instead of assigning the variable to a single value, we assign the variable to a group of values.
  • As long as the number in the variable x is less than the length of the array, the loop will keep running.
  • You put the code that we want to be executed for each loop inside curly brackets.
  • Indices follow a predictable pattern of 0, 1, 2, 3....
  • The code x++ is a shortcut way of writing x+1 because we know each number will always increase by 1 every time the code loops.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • Unlike for loops, functions do not automatically execute when the JS file is run.
  • Being able to call a function to execute it also means you can control where in the control flow a task is executed or if it is executed at all.
  • To create a function we use the keyword "function."
  • The function's name is followed by parentheses.
  • To call a function we use the name of the function in our JS file where we want the code to execute.
  • Uncaught reference error: topic is not defined is a common JS error that devs face commonly.
  • Make sure when changing variables while adding an array to change the initial variable to match the one in the array so that the function looks for the array instead of the variable.
  • Make sure to include [0] with the variable in the array so that it will match the first item in the array to the provided strings.
  • To randomize our selection we use a special "math" function that helps select a random number.
  • In order to have correct control flow, the random variable needs to be AFTER the array is defined.

README

  • In VS code, access README.md to work on the README file.
  • Project title, description, table of contents, installation, usage, credits, license, badges, features, how to contribute, tests are the order of format for a professional README.
  • A README file acts like a virtual storefront to a repo. It's the first thing a person sees when visiting a repo on GitHub.
  • README files contain essential information about the repo's project.
  • There is no right way to structure a README file but there is a wrong way and that is to not include one or create a very sparse one.
  • README files should contain as much information about the repo as pertains such as what was the motivation, why did you build this project and what problem(s) does it solve.
  • README files might also describe challenges faced during the project and features planned for implementation in the future.
  • README files are writted in Markdown and are always named README.md.
